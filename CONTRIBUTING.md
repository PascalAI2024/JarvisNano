# Contributing

JarvisNano is a hardware + firmware + dashboard + Android project. Small,
focused contributions are easiest to review.

## Ground Rules

- Do not commit secrets, Wi-Fi credentials, API keys, private logs, or local
  machine paths.
- Keep generated upstream checkout files out of commits. `esp-claw/` is
  generated by `scripts/bootstrap.sh` and is intentionally ignored.
- Prefer reproducible changes in `boards/`, `firmware/`, `patches/`,
  `scripts/`, `dashboard/`, `android/`, or `docs/`.
- Document hardware-dependent work with the board revision, sensor, wiring, and
  exact test command used.

## Development Setup

Firmware:

```bash
./scripts/bootstrap.sh
./scripts/bootstrap.sh build
./scripts/smoke-build.sh
./scripts/flash.sh
```

Dashboard:

```bash
cd dashboard
python3 -m http.server 8000
```

Android:

```bash
cd android
gradle :app:assembleDebug
gradle :app:testDebugUnitTest
gradle :app:compileDebugAndroidTestKotlin
```

Android builds require JDK 17+ and Android SDK API 35. If using a locally
generated Gradle wrapper, keep generated wrapper scripts and jars untracked
unless the project explicitly decides to commit a trusted wrapper.

## Pull Requests

Before opening a PR, run the checks that match your change:

- Firmware: `./scripts/bootstrap.sh build` and `./scripts/smoke-build.sh`
- Shell/scripts: `bash -n scripts/*.sh`
- Router rules: `jq . firmware/router_rules/router_rules.json`
- Dashboard: JavaScript syntax check or browser reload with console open
- Android: `gradle :app:assembleDebug :app:testDebugUnitTest`

In the PR body, include:

- What changed
- What hardware was tested
- What commands passed
- Any feature that remains blocked by hardware, firmware, or driver behavior
